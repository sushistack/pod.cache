# 002. 🚀 AWS 계정 생성 가이드: 클라우드 여정의 첫걸음

클라우드 컴퓨팅의 선두 주자인 AWS(Amazon Web Services)는 개발자, 스타트업부터 대기업까지 광범위하게 사용되는 강력한 인프라를 제공합니다. 이 아티클은 AWS 클라우드 여정을 시작하려는 모든 분들을 위해, AWS 계정을 안전하고 효율적으로 생성하는 방법을 단계별로 안내합니다.

**3줄 요약:**
*   AWS 계정 생성은 루트 사용자 정보 입력, 강력한 비밀번호 설정, 그리고 계정 플랜 선택으로 진행됩니다.
*   무료(Free) 플랜 선택 시에도 사기 방지 및 본인 확인을 위해 신용카드 정보가 필요하며, $1의 임시 결제 후 즉시 환불됩니다.
*   계정 활성화 후 루트 사용자 이메일로 AWS 관리 콘솔에 로그인하여 클라우드 환경을 안전하게 시작할 수 있습니다.

---

## 1. 서론: 클라우드 컴퓨팅의 문을 열다

현대 소프트웨어 개발과 IT 인프라에서 클라우드 컴퓨팅은 선택이 아닌 필수가 되어가고 있습니다. 그 중심에는 방대한 서비스와 유연성을 제공하는 AWS가 있습니다. AWS를 활용하여 여러분의 아이디어를 현실로 만들고, 더 강력한 애플리케이션을 구축하기 위한 첫 단계는 바로 AWS 계정을 생성하는 것입니다. 이 가이드를 통해 AWS 계정을 안전하게 만들고, 클라우드 환경에서 혁신을 시작할 수 있도록 돕겠습니다.

---

## 2. 본론: AWS 계정 생성 단계별 가이드

AWS 계정 생성 과정은 몇 가지 중요한 단계를 거칩니다. 각 단계를 자세히 살펴보며 주의해야 할 점을 알아보겠습니다.

### 2.1. AWS 계정 생성 시작 및 루트 사용자 정보 입력

AWS 웹사이트에서 '계정 생성(Create an account)' 버튼을 클릭하여 가입 절차를 시작합니다.

#### 2.1.1. 루트 사용자 이메일 및 계정 이름 설정

가장 먼저 **루트 사용자(Root User) 이메일 주소**와 **AWS 계정 이름**을 입력해야 합니다. 루트 사용자 이메일은 계정의 모든 권한을 가지므로, 보안에 각별히 유의해야 합니다.

*   **루트 사용자 이메일**: 개인적으로 자주 사용하는, 보안이 강화된 이메일 주소를 사용합니다.
*   **AWS 계정 이름**: 계정을 식별할 수 있는 고유한 이름을 지정합니다.

<br>

### 2.2. 강력한 비밀번호 설정

루트 사용자 이메일 인증 후, AWS 계정에 사용할 비밀번호를 설정합니다. 이 비밀번호는 계정의 보안에 직결되므로 매우 강력하게 설정해야 합니다.

#### 2.2.1. 비밀번호 정책 준수

AWS는 다음과 같은 비밀번호 정책을 요구합니다.

*   최소 8자 이상
*   다음 네 가지 유형 중 최소 세 가지 포함:
    *   대문자 (Uppercase characters)
    *   소문자 (Lowercase characters)
    *   숫자 (Numbers)
    *   비영숫자 문자 (Non-alphanumeric characters, 예: !@#$%^&*)

**[주의]** 설정한 비밀번호는 절대 잃어버리지 않도록 안전한 곳에 기록하거나 관리해야 합니다.

<br>

### 2.3. 계정 플랜 선택 및 신용카드 정보 입력

AWS는 학습 및 테스트 환경을 위한 '무료 플랜(Free Plan)'과 프로덕션 워크로드를 위한 '유료 플랜(Paid Account)'을 제공합니다.

#### 2.3.1. 무료 플랜 vs. 유료 플랜

두 플랜 모두 초기 크레딧을 제공하지만, 크레딧 소진 후의 처리 방식에 차이가 있습니다.

| 특징           | 무료 플랜 (Free Plan)                                | 유료 플랜 (Paid Account)                                 |
| :------------- | :--------------------------------------------------- | :------------------------------------------------------- |
| 초기 크레딧    | 약 $100 (최대 $200)                                  | 약 $100 (최대 $200)                                      |
| 신용카드 등록  | **필수** (본인 확인 및 사기 방지 목적)               | **필수**                                                 |
| 크레딧 소진 시 | 등록된 신용카드로 **자동 청구 없음**, 계정 기능 제한 | 등록된 신용카드로 **자동 청구**                          |
| 주요 용도      | 학습, 개인 프로젝트, 비생산 환경                     | 운영 환경, 상업적 워크로드, 프로덕션 서비스              |

안전하게 AWS를 학습하기 위해서는 일반적으로 **무료 플랜(Free Plan)**을 선택하는 것이 좋습니다. 만약 무료 플랜만으로는 부족한 유료 서비스가 필요할 경우, AWS가 신용카드 추가를 요청하며 선택에 따라 진행할 수 있습니다.

#### 2.3.2. 신용카드 정보 입력 및 본인 확인

무료 플랜을 선택하더라도 신용카드 정보를 입력해야 합니다. 이는 AWS가 **계정 소유자를 확인하고 사기를 방지**하기 위함입니다. 걱정하지 마세요, 무료 플랜에서는 신용카드 정보 입력만으로는 요금이 부과되지 않습니다.

*   **$1 임시 결제**: 신용카드 유효성 확인을 위해 약 $1이 임시로 결제되지만, 이는 즉시 환불됩니다.
*   **전화번호 인증**: 신용카드 정보 입력 후, 전화번호를 통해 본인 확인 절차를 거칩니다.

<br>

### 2.4. 지원 플랜 선택 및 가입 완료

마지막으로 지원 플랜을 선택합니다. 학습 목적이라면 **기본 지원(Basic Support)**을 선택하는 것으로 충분합니다. 기본 지원은 무료로 제공됩니다.

가입 절차를 완료하면, AWS 계정이 활성화되기까지 잠시 시간이 소요됩니다. 계정 활성화가 완료되면 등록된 이메일 주소로 알림이 발송됩니다.

<br>

### 2.5. AWS 관리 콘솔 첫 로그인

계정 활성화 이메일을 받으면, 이제 AWS 관리 콘솔에 로그인할 수 있습니다.

1.  AWS 웹사이트에서 '콘솔에 로그인(Sign in to console)'을 클릭합니다.
2.  로그인 페이지에서 '루트 사용자 이메일로 로그인(Sign in using root user email)'을 선택합니다.
3.  계정 생성 시 사용했던 루트 사용자 이메일 주소와 비밀번호를 입력합니다.
4.  로그인 시 나타나는 초기 설정 프롬프트(예: MFA 설정 등)는 지금은 건너뛰고 나중에 설정할 수 있습니다.

모든 절차가 완료되면, AWS 관리 콘솔의 대시보드가 나타나며 이제 여러분은 AWS 클라우드 환경에서 서비스를 시작할 준비를 마쳤습니다.
