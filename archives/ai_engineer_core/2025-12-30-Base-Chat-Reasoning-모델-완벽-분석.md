<summary>Base, Chat, Reasoning 모델 완벽 분석</summary>

### 각 모델 유형, 언제 사용해야 할까?

| 모델 유형 | 최적의 사용 사례 | 장점 | 단점 |
| :-------- | :--------------- | :--- | :--- |
| **Base Model** | 새로운 스킬 학습을 위한 모델 훈련 (Fine-tuning) | 기초적인 텍스트 예측 능력, 유연한 훈련 시작점 | 대화형이 아님, 복잡한 태스크에 비효율적 |
| **Chat Model** | 대화형 애플리케이션, 빠른 응답, 창의적 콘텐츠 생성 | 빠르고 저렴함, 자연스러운 대화, 높은 대화 품질 | 복잡한 논리적 추론에서 성능 저하 가능성 |
| **Reasoning Model** | 복잡한 문제 해결, 논리적 추론, 퍼즐, 높은 정확도 요구 | 심층적인 사고 능력, 높은 문제 해결 정확도 | 느리고 비용이 비쌈 (추론 토큰으로 인한), 과도한 추론으로 인한 "과잉 분석" 가능성 |
| **Hybrid Model** | 다양한 난이도의 태스크를 처리해야 하는 다목적 시나리오 | 필요에 따라 유연하게 Reasoning Budget 조절, Chat과 Reasoning의 균형 | 내부 작동 방식의 복잡성 |
